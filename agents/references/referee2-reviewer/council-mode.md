# Referee 2 — Portable Council Mode

The main session may run the five-audit referee protocol through a verified
external council backend. Read the installed shared `council-protocol.md`.

1. Assemble the audit instructions, paper, code, and reproducibility evidence.
2. Run all local hard gates before any paid model call.
3. Use `council-api` only after explicit user approval and a successful
   `council-api --help` preflight.
4. Require at least two independent model assessments before peer review and
   chairman synthesis.
5. Render the result in the normal referee report format with council metadata.

Provider-specific CLIs are optional user-installed integrations. This public
workflow never assumes a private package checkout or a client-specific home.
